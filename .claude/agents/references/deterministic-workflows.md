# Deterministic AI agent workflows

**Canonical reference.** Skills that use these scripts (verify-task, document, document-agents) link here; do not duplicate this content in skills.

We use **TypeScript scripts as the source of truth** so the same input consistently produces the same output. Workflow and structure are fixed; content stays flexible. Scripts live in the repo so logic is reviewable and maintainable.

## Principles

1. **Consistent input/output** – Same input → same output. Flow choice, step list, doc outline, and subagent choice come from scripts, not from the model. That removes variability in decisions.

2. **Script execution** – Run scripts via npm and pass input on the command line. Example: `npm run checklist -- "<request>"` or `npm run doc:structure -- project-overview`. Predictable result every time.

3. **Version control** – Scripts live in the repo. Logic is consistent, reviewable, and maintainable over time. Change scripts in code; no hidden or prompt-only behavior.

4. **Fixed logic for choices** – Use dedicated scripts for anything that must be consistent: flow/steps (checklist), doc section order (doc-structure), subagent choice (pick-subagent). Workflow stays stable regardless of model output.

5. **Flexibility in content** – Structure and workflow are fixed; the actual text, diagrams, and content are generated by the model. Predictability where it matters, freedom where it helps.

6. **Feedback loop** – Review and refine scripts based on use. When outcomes drift or new cases appear, update the script and commit. Scripts improve over time like any other code.

## Scripts (source of truth)

| Script | npm command | What it fixes |
|--------|-------------|---------------|
| Checklist | `npm run checklist -- "<request or summary>"` | Flow and step list for the task |
| Doc structure | `npm run doc:structure -- <document-type>` | Section outline for README/supplementary docs |
| Pick subagent | `npm run doc:pick-subagent -- "<task message>"` | Which doc subagent (explore, plan, general-purpose, main) |

Same request or document type → same flow, same outline, same subagent. The model then fills in content within that structure.
